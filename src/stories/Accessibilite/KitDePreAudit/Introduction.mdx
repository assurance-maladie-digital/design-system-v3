# Pré-audit web conformité à l’accessibilité

## Objectif

L’objectif du pré-audit est d’obtenir un premier avis sur le taux de conformité de votre service web avant de convenir d’un réel audit RGAA. La procédure se fait en autonomie par l’équipe de développement.
#
La restitution consiste à s’assurer que toutes les exigences du pré-audit sont remplies et que vous ayez un minimum d’erreur en lançant les tests automatisés avec Tanaguru.
Vous pouvez nous solliciter pour un accompagnement si vous rencontrez des difficultés techniques à résoudre les non-conformités relevées par Tanaguru.

## Environnement de test

- Système d’exploitation : Windows
- Navigateur : Firefox (mentionner la version)
- Lecteur d’écran : NVDA (dernière version)

Si l’outil cible un public utilisant des terminaux mobile, utiliser l’environnement de test suivant en supplément :

- Système d’exploitation : Android
- Navigateur : Chrome (mentionner la version)
- Lecteur d’écran : TalkBack (dernière version)

## Livrable

- La Check-list du pré-audit qui comprend :
  - contexte du projet (responsabilités)
  - résultat de l’échantillon de test
  - résultat du pré-audit manuel complété avec 1 onglet / page
- En annexe et pour chaque page, il sera demandé de restituer un rapport Tanaguru
